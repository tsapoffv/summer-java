# Морской бой (Battleship)

Клиент-серверная реализация классической игры «Морской бой» на Java с графическим интерфейсом Swing и текстовым сетевым протоколом.

```bash
javac server/*.java
java server.Server
```

### Клиент

```bash
javac client/*.java
```

## 🌐 Сетевой протокол

Протокол **текстовый**, строки передаются через `DataInputStream.readUTF()` / `DataOutputStream.writeUTF()`.

| Команда (направление)                           | Описание                                      |
|------------------------------------------------|-----------------------------------------------|
| `CMD_REQUIRE_FIELD` (Сервер → Клиент)           | Сервер запрашивает поле                       |
| `SET_FIELD:<100×0/1>` (Клиент → Сервер)         | Отправка поля (строка 10×10 = 100 символов)   |
| `CMD_OK:Field accepted` (Сервер → Клиент)       | Поле принято                                  |
| `CMD_ERROR:<текст>` (Сервер → Клиент)           | Ошибка с описанием                            |
| `GET_PLAYERS` (Клиент → Сервер)                 | Запрос списка игроков                         |
| `PLAYERS:<id1>,<id2>,...` (Сервер → Клиент)     | Список ID готовых игроков                     |
| `SELECT_PLAYER:<id>` (Клиент → Сервер)          | Выбор противника по ID                        |
| `CMD_GAME_START:<info>` (Сервер → Клиент)       | Игра началась                                 |
| `CMD_YOUR_TURN` (Сервер → Клиент)               | Ваш ход                                       |
| `CMD_OPPONENT_TURN` (Сервер → Клиент)           | Ход противника                                |
| `MOVE:<x>:<y>` (Клиент → Сервер)                | Выстрел по координатам                        |
| `CMD_MOVE_RESULT:<r>:<x>:<y>` (Сервер → Клиент) | Результат: `MISS` / `HIT` / `SUNK` / `WIN`    |
| `CMD_OPPONENT_MOVE:<r>:<x>:<y>` (Сервер → Клиент) | Противник выстрелил                         |
| `CMD_AGAIN` (Сервер → Клиент)                   | Попал, ходи ещё                               |
| `CMD_GAME_OVER:<WIN/LOSE>` (Сервер → Клиент)    | Игра окончена                                 |
| `SURRENDER` (Клиент → Сервер)                   | Сдаться                                       |

## 🖥 Серверная часть

### `Server.java` – главный класс

- **Назначение:** точка входа, приём подключений, управление клиентами, создание игр.
- **Поля:**
  - `ConcurrentHashMap<Integer, ClientHandler> clients`
  - `AtomicInteger nextId`
  - `static final int PORT = 12345`
  - `static final int[] SHIP_SIZES = {4,3,3,2,2,2,1,1,1,1}`
- **Методы:**
  - `main()` → `new Server().start()`
  - `start()`: `ServerSocket.accept()` в цикле, каждый клиент передаётся в `newCachedThreadPool`.
  - `synchronized startGame(ClientHandler a, ClientHandler b)`: проверка готовности, создание `Game`.

**Почему `newCachedThreadPool`?** Динамическое число клиентов, потоки переиспользуются.  
**Почему `synchronized` на `startGame`?** Защита от одновременного вызова двумя клиентами.  
**Почему `ConcurrentHashMap`?** Потокобезопасная итерация без блокировки всей карты.

---

### `ClientHandler.java` – обработчик клиента

- **Назначение:** один экземпляр на подключение, работает в отдельном потоке (`Runnable`).
- **Поля:**
  - `Socket socket`
  - `DataInputStream in` / `DataOutputStream out`
  - `int id`
  - `Server server`
  - `boolean[][] myField`
  - `boolean[][] opponentHits`
  - `Game currentGame`
  - `boolean ready`
- **Жизненный цикл:**
  1. **Получение поля** (`receiveField`): цикл запросов `CMD_REQUIRE_FIELD`, валидация через `FieldValidator.validate()`, ответ `CMD_OK` или `CMD_ERROR`.
  2. **Обработка команд** (`processCommands`): бесконечный цикл чтения команд:
     - `GET_PLAYERS` → `sendPlayerList()`
     - `SELECT_PLAYER:` → `selectOpponent()`
     - `MOVE:x:y` → `makeMove()`
     - `SURRENDER` → `surrender()`
  3. **Завершение** (finally): закрытие сокета, завершение игры, удаление клиента.

---

### `Game.java` – логика игры

- **Назначение:** одноразовый объект для одной партии.
- **Поля:** `ClientHandler p1, p2`, `int currentTurn`, `boolean active`.
- **Методы:**
  - `makeMove(mover, x, y)`:
    1. Определяет противника (`opp`).
    2. Проверяет попадание: `hit = opp.myField[x][y]`.
    3. Записывает выстрел в `opponentHits`.
    4. Определяет результат (`MISS`, `HIT`, `SUNK` через `isShipSunk()`).
    5. Проверяет победу (`checkWin`).
    6. Отправляет результаты обоим игрокам.
    7. При попадании даёт дополнительный ход (`CMD_AGAIN`), иначе переключает ход.
  - `isShipSunk(field, hits, x, y)`: DFS по полю, находит все клетки корабля и проверяет, все ли подбиты.
  - `checkWin(opp, hits)`: проверяет, остались ли живые клетки у противника.
  - `endGame(loser, surrender)`: завершает игру, отправляет `CMD_GAME_OVER`.

---

### `FieldValidator.java` – валидация поля

- **Назначение:** статический утилитарный класс для проверки корректности расстановки.
- **Алгоритм `validate(field)`:**
  1. Поиск всех кораблей через DFS (`findShipCells`).
  2. Для каждого корабля:
     - размер от 1 до 4;
     - прямолинейность (`isShipStraight`);
     - отсутствие касаний с другими кораблями (`hasNoTouching`);
  3. Подсчёт количества кораблей каждого размера и сравнение с эталоном `[1,2,3,1]` (1×4, 2×3, 3×2, 4×1).
- **Почему отдельный класс?** Чистая функция, не зависит от состояния сервера, легко тестируется.

## 🎨 Клиентская часть

### `BattleshipClientGUI.java` – главное окно

- **Назначение:** `JFrame`, управление экранами через `CardLayout`, сетевое соединение, диспетчер сообщений.
- **Компоненты:**
  - `CardLayout cardLayout`, `JPanel mainPanel`
  - `Socket socket`, `DataInputStream in`, `DataOutputStream out`
  - `Thread receiverThread` (daemon)
- **Сетевая модель:**
  - `connect(host, port)`: запускает поток, создаёт сокет, инициализирует потоки, запускает `receiverThread`.
  - `receiveLoop()`: цикл чтения, каждое сообщение передаётся в `handleMessage()` через `SwingUtilities.invokeLater()`.
- **`handleMessage()` – диспетчер:**
  - `CMD_REQUIRE_FIELD` → переход в `SetupPanel`
  - `CMD_OK` → переход в `LobbyPanel`
  - `PLAYERS:` → обновление списка в лобби
  - `CMD_GAME_START:` → переход в `BattlePanel`
  - `CMD_YOUR_TURN` / `CMD_OPPONENT_TURN` → установка флага хода
  - `CMD_MOVE_RESULT:` / `CMD_OPPONENT_MOVE:` → обновление полей
  - `CMD_AGAIN` → уведомление о доп. ходе
  - `CMD_GAME_OVER:` → завершение игры, переход в лобби
- **Почему `SwingUtilities.invokeLater()`?** Все манипуляции с GUI должны выполняться в EDT.

---

### `ConnectionPanel.java` – экран подключения

- **Назначение:** форма ввода хоста/порта, кнопка подключения, индикатор статуса.
- **Компоновка:** `GridBagLayout` для центрирования.
- **Взаимодействие:** кнопка вызывает `client.connect()`.

---

### `SetupPanel.java` – расстановка кораблей

- **Назначение:** ручная или автоматическая расстановка 10 кораблей.
- **Состояние:** `boolean[][] field`, `List<Integer> remainingShips`, `selectedShipSize`, `horizontal`.
- **Структура:** `BorderLayout` с панелью поля (`BoardPanel` режим `SETUP`) и палитрой кораблей.
- **Механика расстановки:**
  - Клик по кораблю в палитре → выбор размера.
  - Наведение мыши → предпросмотр (зелёный / красный).
  - Клик по полю → проверка `canPlace()` (границы и касания), размещение, обновление палитры.
- **Авторасстановка:** случайная генерация с повторными попытками (до 5000 на корабль), при неудаче – рекурсивный перезапуск.
- **Горячая клавиша R** – поворот (реализована через `InputMap`/`ActionMap`).

---

### `LobbyPanel.java` – лобби

- **Назначение:** список доступных игроков, вызов на бой.
- **Компоненты:** `JList<PlayerItem>` с кастомным рендерером, кнопки «Обновить» и «Вызвать».
- **Flow:** `GET_PLAYERS` → обновление списка, выбор игрока → `SELECT_PLAYER`.

---

### `BattlePanel.java` – экран боя

- **Назначение:** игровой процесс – два поля, статус, лог, кнопка сдачи.
- **Структура:** `BorderLayout` с панелью статуса, двумя `BoardPanel` (своё и врага), `EventLog` и кнопкой «Сдаться».
- **Состояние клеток (int):**
  - `0` – вода / неизвестно
  - `1` – корабль
  - `2` – попадание
  - `3` – промах
- **Клик по полю противника:** проверка `inGame && myTurn`, отправка `MOVE:x:y`.
- **Обновление:** `applyMyMove()` / `applyOpponentMove()` обновляют массивы и логи.

---

### `BoardPanel.java` – кастомная отрисовка поля

- **Назначение:** рендеринг игрового поля с анимациями, поддержка трёх режимов (`SETUP`, `MY_FIELD`, `ENEMY_FIELD`).
- **Рендеринг клеток (Java 2D):**
  - Вода – градиент `WATER_MID → WATER_DEEP`.
  - Корабль – градиент `SHIP_HIGHLIGHT → SHIP_BASE`, скруглённые углы.
  - Попадание – красный градиент, 8-лучевая звезда.
  - Промах – вода с синим кольцом и точкой.
- **Анимации (Swing Timer, 60 FPS):**
  - `EXPLOSION` – расширяющиеся кольца + искры.
  - `SPLASH` – водяные круги.
- **Подсветка при расстановке:** зелёный / красный полупрозрачный прямоугольник.
- **Координатные метки:** номера строк и столбцов по краям.

---

### Вспомогательные компоненты

- **`GradientButton`** – кнопка с градиентной заливкой, эффектами наведения и нажатия.
- **`RoundedTextField`** – текстовое поле со скруглёнными углами и рамкой, меняющей цвет при фокусе.
- **`EventLog`** – лог событий с цветовой кодировкой (`SYSTEM`, `INFO`, `SUCCESS`, `WARNING`, `DANGER`), автоматическая прокрутка.
- **`PlayerItem`** – простая модель данных для списка игроков.
- **`Constants`** – игровые константы (`BOARD_SIZE`, `CELL_SIZE`, `SHIP_SIZES`).
- **`Colors`** – единая цветовая схема (тёмная тема).
